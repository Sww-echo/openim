# 2026-06-20 手机号参数前置校验统一

## 背景

- 当前登录、注册、找回密码均收敛为手机号流程，区号列表仅提供 `+86`。
- `/account/login`、`/account/register`、`/account/code/send`、`/account/code/verify` 均依赖有效手机号参数。
- 注册页已有 `^1\d{10}$` 校验，但登录页和忘记密码页此前只校验必填，仍可能把明显非法手机号发送给业务接口。

## 调整

- 新增 `src/pages/login/rules.ts`，统一手机号规则：必填 + `^1\d{10}$`。
- 登录页手机号字段接入统一规则，避免向 `/account/login` 提交非法手机号。
- 注册页手机号字段接入统一规则，并移除提交函数中的重复手工校验。
- 忘记密码手机号字段接入统一规则，发送验证码前的 `validateFields(["areaCode", "phoneNumber"])` 会同时阻断空值和格式错误，避免向 `/account/code/send` 提交非法手机号。

## 验证

- 本轮只做源码静态复核和登录页 HTTP 只读复核。
- 未运行单元测试、构建或验证脚本。
- 未触发真实登录、注册、验证码发送、验证码校验或重置密码请求。
