# 2026-06-20 登录入口收敛到手机号密码

## 背景

继续对照本地 `docs/openim-swagger.json` 复核登录接口时，确认 `/account/login` 最新摘要为：

> 账号登录。兼容前端JSON请求，当前阶段使用手机号和密码登录

页面此前仍保留手机号验证码登录切换，会把 `verifyCode` 直接随 `/account/login` 提交。Swagger 当前没有定义验证码登录参数或验证码登录专用登录接口，这会让登录入口发送未声明字段，并且可能缺少密码字段。

## 本次处理

- `src/pages/login/LoginForm.tsx`
  - 移除登录页验证码登录切换、验证码输入和登录页发送验证码逻辑。
  - 登录页只保留手机号、密码、固定企业号字段。
  - 手机号和密码增加表单必填校验，避免向 `/account/login` 发送空手机号或空密码。

## 接口影响

- `/account/login`：当前只从登录页发送手机号 + MD5 密码 + 固定企业号相关字段，符合“当前阶段使用手机号和密码登录”的接口说明。
- `/account/code/send`、`/account/code/verify`：注册和找回密码流程仍继续使用，本次未修改。

## 验证状态

本轮未运行单元测试、构建或验证脚本，未新增或修改测试文件。后续使用真实浏览器只读打开登录页确认页面渲染；不会提交登录请求。
