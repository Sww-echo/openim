# 2026-06-25 用户目标重申

## 最新目标

用户明确要求：创建新的 Trellis 任务，用于继续实现当前 Web 端尚未完成的接口调用。

该目标是在既有 `web-client-api-integration` 任务基础上的后续拆分推进，重点不再只是“统计/总结/静态对齐”，而是继续把已识别但尚未落到页面或调用链的 Web 用户端接口接入到当前项目中。

## 执行授权

用户明确说明当前环境使用的是测试数据，并授权：

- 可以真实调用后端接口。
- 可以在测试数据上触发写操作或修改数据。
- 可以用浏览器复测 mutation 类接口，不再受此前“不要主动触发写操作”的限制。

## 执行边界

- 仍以 Web 用户端为范围，不主动接入后台、支付、运营、运维桥接等非 Web 用户端接口。
- 接口来源仍以 `docs/openim-swagger.json` 和当前代码实现为准。
- 优先处理当前已经在 Trellis 中标记为未完成、暂缓或待后端契约确认但现在允许真实联调的接口。
- 后续实现时需要同步更新新的 Trellis 任务记录，明确哪些接口已接、已验证、失败原因和剩余项。

## 首批候选方向

- `/user/logout`：此前因缺少稳定 `deviceKey/devicekey/telephone(MD5)` 来源未强接；现在可重新联调真实退出接口契约。
- `/room/add`：此前因 Swagger 契约偏旧系统复杂结构未替换 SDK 建群；现在可创建独立任务继续确认并尝试接入业务建群调用。
- 其它当前 Trellis 中归为“待产品入口/后端契约明确后再接”的 Web 用户端接口，可逐项复核后纳入新任务。
