## 执行顺序

1. 登录功能：先保证业务 token、OpenIM token、账号隔离、IP 限制和宵禁提示稳定。
2. 聊天功能：在登录态稳定后处理消息发送、文件、搜索、收藏、合并转发、群设置/成员只读查看。
3. 群管理功能：最后处理群公告、群成员、群权限、消息销毁、邀请审核、已读详情、在线成员等高副作用或依赖 business roomId 的能力。

## 通用执行规则

- 每个功能项都必须按 `docs/openim-frontend-api-doc.json` 核对接口。
- 旧接口如果不在新文档中，必须替换为新文档接口；没有新文档等价接口的，明确记录为文档缺口或 SDK 承担。
- 当前测试环境允许真实调用接口、触发写操作、修改测试数据；普通测试写操作不再逐次确认。
- 每完成一个大功能项，必须在对应子任务目录新增总结统计记录。
