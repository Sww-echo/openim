# Web 用户端接口接入覆盖梳理

## 背景

Web 用户端需要覆盖登录、聊天、文件预览下载、群管理等核心 IM 工作流。当前后端 Swagger 已保存到 `docs/openim-swagger.json`，文档包含 780 个唯一 path。本文档把产品功能拆成前端接入所需的接口包，避免把后台管理、支付、运营等非首期能力混入 Web 用户端范围。

## 功能范围

### 2.1 登录功能

需要覆盖：

- 企业号验证
- 个人账号登录
- Web 端多账号保存
- 多账号一键切换
- 不同账号数据隔离
- IP 限制登录提示
- 宵禁登录限制提示

需要接口分类：

- `企业号注册`
- `用户账号登陆注册相关操作  新接口`
- `OpenIM-账号Token`
- 登录安全预检/限制提示接口，当前文档主要在 `console` 后台安全分类下，需要后端确认是否提供用户端版本或由 `/account/login` 返回明确错误码。

前端职责：

- 多账号保存、一键切换和数据隔离主要由 Web 本地状态、token 存储、OpenIM SDK 登录目录/实例隔离实现。
- 切换账号时必须切换业务 token、OpenIM token、OpenIM userID，并重新初始化 SDK 会话。

### 2.2 聊天功能

需要覆盖：

- 群聊列表
- 单聊列表
- 群消息收发
- 文件上传下载
- 图片/视频预览
- 聊天记录搜索
- 消息转发、复制、删除
- 群设置查看
- 群成员查看，受权限控制

需要接口分类：

- `OpenIM-单聊桥接`
- `OpenIM-消息发送`
- `OpenIM-群聊桥接`
- `OpenIM-文件上传`
- `OpenIM-消息收藏`
- `OpenIM-合并消息`
- `好友管理相关接口`
- `群组接口`

实现边界：

- 会话列表、普通单聊/群聊消息收发优先走 OpenIM SDK。
- 发送前业务校验、聊天记录搜索、群消息撤回、文件签名/预览/下载需要业务 API。
- 复制属于纯前端能力；删除需要按产品语义确认是本地删除、服务端删除还是消息撤回。

### 2.3 群管理功能

需要覆盖：

- 群主/管理员管理入口
- 群公告管理
- 群成员管理
- 群权限设置
- 消息销毁设置
- 邀请审核
- 查看已读详情
- 查看在线成员

需要接口分类：

- `OpenIM-群聊桥接`
- `群组接口`
- `OpenIM-群助手`，仅在需要群助手/自动回复时接入
- `用于组织架构功能的相关接口`，仅在群权限依赖企业组织角色时接入

实现边界：

- 群详情、成员列表、在线成员、入群审核、群公告、群免打扰、群置顶、消息游标清理等使用业务 API。
- 群主/管理员入口是否显示，应以前端已有 OpenIM 群角色信息和后端接口返回权限共同判断。

## 非首期范围

- `console` 后台管理完整能力
- 支付、转账、消费记录
- 商务圈、直播间、客服、商品、运营活动
- 平台总后台和企业后台安全配置管理

这些接口不属于当前 Web 用户端首期功能，除非产品明确要求放到用户端。

## 验收标准

- 登录页能完成企业号校验、账号登录，并正确处理 IP 限制和宵禁提示。
- 多账号切换后，业务 token、OpenIM token、OpenIM userID 与本地数据隔离正确。
- 群聊/单聊列表和消息收发走通，发送前校验失败能给出明确提示。
- 文件上传、下载、图片/视频预览走通。
- 单聊和群聊聊天记录搜索可用。
- 群详情、成员列表、公告、邀请审核、已读详情、在线成员等群管理入口按权限展示并可用。

## 待确认问题

- Swagger 中未看到当前前端已有的 `/user/find/full` 和 `/user/search/full`，需要确认是否已移除，或改用 `/friends/get`、`/friends/list`、`/room/member/get` 等接口替代。
- IP 限制和宵禁相关接口目前主要在 `console` 后台分类下，Web 用户端是否直接使用这些接口，需要后端确认。
- “消息删除”需要确认产品语义：本地删除、服务端删除、群消息撤回或聊天记录审计删除。
- “群权限设置”和“消息销毁设置”的最终接口需结合后端字段定义确认，Swagger 里存在群免打扰、置顶、清空消息游标、特殊成员等能力，但未形成统一的用户端权限设置接口。
