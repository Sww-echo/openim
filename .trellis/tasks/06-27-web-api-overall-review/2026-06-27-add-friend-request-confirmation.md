# 添加好友打招呼流程排查记录

## 问题

- 用户卡片点击添加好友后，进入打招呼消息页后还会再弹出“确认发送好友申请吗”二次确认。
- 被添加账号在另一个浏览器页面没有收到新朋友申请提醒，而是直接在好友列表出现该好友。

## 调整

- 保留用户卡片的打招呼消息页。
- 移除打招呼消息页内的二次确认弹窗，点击发送直接提交。
- `/friends/add` 调用补充可选 `reqMsg` 参数，非空时随请求带上打招呼内容。

## 原因判断

- 最新接口文档中 `/friends/add` 的语义是“添加好友”，参数只声明 `toUserId`，没有“提交申请/等待同意”的开关或申请理由字段。
- 当前后端行为会直接建立好友关系，因此另一个浏览器会收到好友关系变化并进入好友列表。
- “新的好友”提醒依赖好友申请事件或新朋友申请列表；当前 `/friends/add` 直接加好友时不会产生待处理申请。

## 后续接口缺口

- 如果产品要求被添加方先收到申请提醒并手动同意，需要后端提供“发送好友申请”的接口，或调整 `/friends/add` 在需要验证时创建申请而不是直接建立好友关系。
